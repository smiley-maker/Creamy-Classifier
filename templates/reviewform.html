creamy classifier

We offer a dynamic platform for ice cream reviews. The reviews are evaluated using machine learning based sentiment analysis to determine if they are more positive or negative. The classifier also employs a continuous learning approach by asking for your feedback!

Learn More About Us!

Creamy Classifier allows customers to review their experience with Ben & Jerry’s ice cream. It is a useful business tool because the company can get an overall estimate of how a large amount of customers feel about their products. The classifier rates reviews as either generally more positive or negative. Further, customers are able to indicate whether this sentiment classification was correct or not, providing a continually learning model.

How it works:

Stochastic Gradient Descent Classification

Creamy Classifier uses a sochastic gradient descent optimization technique along with a linear support vector machine model. Gradient descent in general is often used in minimization problems, in which a target goal is desired, such as minimizing the cost of some series of actions. Stochastic gradient descent is similar, but allows the gradient to be taken with just one sample.

Count Vectorizer

The classifier also implements a count vectorizer which is used to transform a given review into a vector. It uses the frequency of each word in the review to creae a kind of table containing only unique words. This makes it much easier for machine learning models to understand and classify the text.

TFIDF Vectorizer

The Term Frequency Inverse Document Frequency vectorizer is used to transform text into meaningful numbers that the model can understand. It is essentially a measure of how unique a word is. The mathematical component to the TFIDF vectorizer is more complex, but it involves using the associated word weights described by the count vectorizer.

Pickling the model

Out of core learning was implemented in order for the model to be pickled and used in the web based review classification system. Pickling involved serializing the model so that a standalone Python model was created implementing a hashing vectorizer using the optimized tfidf vectorizer. A SQLlite database was created so that the machine learning model could be continuously updated with new data from reviewers.

Website Creation

Finally, Flask was used to link the machine learning model to various components on a (this) website.
